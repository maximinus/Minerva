We need base SDL capability, but not much of it

We render the GUI in layers. If we imagine a list of layers:

    [a, b, c]

Then they will drawn in the order a -> b -> c, but mouse checks will perform in the other direction

Because of this,the method of execution in a frame is:

    Build the sizes of the widgets for each layer
        This can be avoided by caching the result, but then we need to suggest we have changed

    Check where the mouse is over now and adjust
        Set UI items active or elsewise; because we are immediate some status checks not required
    
    Render the widgets
        Which COULD also be cached, but be a lot more wary here maybe.
