
Let's start with the most basic, a ColorRect, almost always used for testing. It has:

A min size
It can be set to expand in one direction or the other
If there is more space than required, it has a align to show how it should be aligned
It has a color

And that's it!

Now, do we add a margin to that, or not?

If it has a margin, then the only REAL effect is that:

The min size must be recomputed
The drawing must at some offset
Since there is an area AROUND the widget, it must have some color, or be see-through
A margin makes more sense as a programming concept, and CSS has this


Now we have a ColorRect, we need to have some more things

A VBox and a HBox, for holding things

For example, a HBox needs the following:

A list of children
A value indicating space between the objects

And that's it!

We also want a SPACER object, this is like a ColorRect, but with no color


We need a label:

Min Size
Expand
Align
Text
